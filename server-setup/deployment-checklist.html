AWS SES — nastavení
Ověření odesílací identity
IAM uživatel pro API
IAM inline policy
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": "ses:SendEmail",
    "Resource": "*"
  }]
}
Výstup ze sandboxu
⚠ Hetzner blokuje port 25 (SMTP), ale AWS SES SDK používá HTTPS (port 443) — blokáda se netýká. Odesílání přes API pojede bez problémů.

Server — přihlášení do GHCR
ℹ Potřebné pouze pokud je GitHub Container Registry package privátní. Veřejný package nevyžaduje přihlášení.
Příkaz na serveru
echo "GITHUB_PAT" | docker login ghcr.io -u GITHUB_USERNAME --password-stdin
✓ Credentials se uloží do ~/.docker/config.json — přihlášení přetrvá, opakovat není třeba.
6
Server — docker-compose.yml a .env
Kopírování souborů (Total Commander + SFTP)
Obsah .env — co vyplnit
ConnectionStrings__DefaultConnection=Host=...;Database=...;Username=...;Password=...
ApiKey=                        ← náhodný bezpečný řetězec (min. 32 znaků)
Aws__Region=eu-central-1
Aws__AccessKeyId=              ← z IAM uživatele
Aws__SecretAccessKey=          ← z IAM uživatele
Email__FromAddress=            ← ověřená adresa v SES
Email__FromName=               ← zobrazované jméno odesílatele
Email__ReplyToAddress=         ← kam odpovídat
Email__UserPortalBaseUrl=      ← URL frontendu (https://...)
Email__DryRun=false
Email__OverrideRecipient=      ← prázdné = produkce, nebo testovací email
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://+:8080
⚠ .env nikdy necommitovat do gitu. Do gitu patří jen .env.template s prázdnými hodnotami.
7
Watchtower — automatický deploy
ℹ Watchtower sleduje běžící kontejnery s příslušným labelem a automaticky je aktualizuje při detekci nového :latest image v registry.
Spuštění Watchtower (pokud ještě neběží)
docker run -d \
  --name watchtower \
  --restart unless-stopped \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v ~/.docker/config.json:/config.json \
  containrrr/watchtower \
  --interval 300
